Functions of Operating System

Delhi Police Exam

Introduction to OS Functions

Understanding the functions of an Operating System (OS) is key for Delhi Police Computer Awareness exams.

Questions often target specific functions, their examples, or related concepts.

Exam Insight: Focus on understanding each function's purpose and real-world examples as Delhi Police frequently tests application-based knowledge.

1. Main Functions of an Operating System

1

Memory Management

  • Allocates RAM to running programs
  • Keeps track of used and free memory
  • Ensures no program interferes with another

Example: Virtual memory usage in Windows

2

Process Management

  • Manages execution of programs (processes)
  • Handles CPU scheduling, multitasking, and process coordination

Example: Task Manager in Windows shows running processes

3

File Management

  • Organizes files and folders on storage devices
  • Handles read/write operations, file creation, deletion, and access rights

Example: File Explorer in Windows or Finder in macOS

4

Device Management

  • Manages input/output devices like keyboard, mouse, printer, scanner
  • Ensures proper communication between devices and CPU

Example: Printer driver handling print commands

5

Additional Functions (Optional but Useful for MCQs)

Security & Access Control

Protects data from unauthorized access

User Interface

GUI or CLI for user interaction

Error Detection & Handling

Identifies and corrects errors during processing

4. Quick Recap & Memory Tricks

Memory Management → RAM allocation

Process Management → CPU scheduling, multitasking

File Management → Organizes files & folders

Device Management → Handles input/output devices

Memory Trick: "Memory → RAM, Process → CPU, File → Folders, Device → Peripherals"

Additional Functions: Security, User Interface, Error Handling

Exam Tip: Focus on examples of each function; questions often appear as direct MCQs or scenario-based. Practice matching functions with their real-world examples.
